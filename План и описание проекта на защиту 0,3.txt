Презентация проекта "Онлайн-викторина" (C#, .NET Framework, Windows Forms)**

1) ПРИВЕТСВИЕ
всем здравствуйте, рады вас здесь видеть

2) ПРЕДСТАВЛЕНИЕ
сегодня мы бы хотели вам представить наш проект игра-викторина, которая называется Wikviz

3) ВВЕДЕНИЕ
в этом игре два участника соревнуются в реальном времени, отвечая на вопросы, хранящиеся в базе данных. Приложение состоит из сервера, который управляет игрой, и клиентов, подключающихся к нему через IP Adres Сервера и его Порта.  

4) ПРОЦЕСС
Сервер стартует и ожидает подключений через TCP/IP. Как только оба игрока присоединяются, он загружает 3 случайных вопроса из базы данных (всего 12) (например, SQLite или MS SQL) и рассылает их клиентам. Таймер начинает отсчет, и игроки отвечают через интерфейс Windows Forms. Сервер проверяет корректность ответов, ведет счет и объявляет результат в конце раунда

5) ОСНОВА
### *Технические детали*  
- Для сетевого взаимодействия используется *System.Net.Sockets* (или SignalR для более высокоуровневой реализации).  
- Данные хранятся в реляционной БД, а ADO.NET обеспечивает быстрое подключение и выполнение запросов.  
- Интерфейс клиента интуитивно понятен: вопросы выводятся в Label, ответ записывается в text box и отправляет рузультат с помощью кнопки, а таймер отображаются в реальном времени в углу экрана.  

6) СИСТЕМА
Весь код был написан на языке C#
Сервер - Консольное приложение (NET Framework)
Клиент - Приложение Windows Forms NET (Framework)
База данных с таблицей из 12 строчек и 3 столбцов - SQL
### *Ключевые особенности*:
1. *Сервер на C#* обрабатывает подключения игроков, загружает вопросы из БД через ADO.NET и синхронизирует начало викторины.  
2. *Клиенты* получают одинаковые вопросы одновременно, а ответы отправляют обратно на сервер. Побеждает тот, кто даст больше правильных ответов за 1 минуту.  

7) В БУДУЩЕМ
В будущем мы могли бы добавить систему регистрации, чат между игроками и возможность состязатья более двум игрокам, или одному, проверить свои знания. Также можно расширить функционал, добавив разные категории вопросов и уровни сложности.
