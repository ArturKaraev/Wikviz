Презентация проекта "Онлайн-викторина" (C#, .NET Framework, Windows Forms)**

1) ПРИВЕТСВИЕ
всем здравствуйте, рады вас здесь видеть

2) ПРЕДСТАВЛЕНИЕ
сегодня мы бы хотели вам представить наш проект игра-викторина, которая называется Wikviz

3) ВВЕДЕНИЕ
в этой игре два участника соревнуются в реальном времени, отвечая на вопросы, хранящиеся в базе данных. Приложение состоит из сервера, который управляет игрой, и клиентов, подключающихся к нему через его IP Adres и Порт.  

4) ПРОЦЕСС
Как происходит процесс самой игры? Сервер стартует и ожидает подключения через TCP/IP. Как только оба игрока присоединяются, он загружает 3 случайных вопроса из базы данных (всего их 12) (например, SQLite или MS SQL) и рассылает их клиентам. Таймер начинает отсчет, и игроки отвечают через интерфейс Windows Forms отвечают на вопросы по порядку (у каждого клиента одинаковые вопросы и вопрос можно пропустить по желанию, а также можно сдатья, тогда игрок проиграет). Сервер в свою очередь проверяет корректность ответов, ведет счет и объявляет результат в конце раунда

5) ОСНОВА
### *давайте теперь расскажем вам о Технических деталях*  
- Для сетевого взаимодействия мы используем *System.Net.Sockets* (или SignalR для более высокоуровневой реализации).  
- Данные хранятся в реляционной БД, а ADO.NET обеспечивает быстрое подключение и выполнение запросов.  
- Интерфейс клиента интуитивно понятен: вопросы выводятся в Label, ответ записывается в text box и отправляется рузультат с помощью кнопки, а таймер отображаются в реальном времени в углу экрана.  

6) СИСТЕМА
теперь поговорим о системе 
система состоит из:
Сервера - Консольное приложение (NET Framework)
Клиента - Приложение Windows Forms (NET Framework)
Базы данных с таблицей из 12-ти строчек и 3-х столбцов (в SQL сервере)
а ещё, что хотелось бы сказать, это то что Весь код был написан на языке C#
### *Ключевые особенности*:
1. *Сервер* обрабатывает подключения игроков, загружает вопросы из БД через ADO.NET и синхронизирует начало викторины.  
2. *Клиенты* получают одинаковые вопросы одновременно, а ответы отправляют обратно на сервер. Побеждает тот, кто даст больше правильных ответов за 1 минуту.  

7) В БУДУЩЕМ
